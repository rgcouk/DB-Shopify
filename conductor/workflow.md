# Workflow - DB-Shopify

## TDD Policy
**Moderate**: Tests are encouraged, especially for core logic and shared UI components.

## Commit Strategy
**Squash commits per task/feature**: Maintain a clean history by squashing development commits into a single logical unit.

## Code Review Requirements
**Optional / self-review OK**: Direct pushes are allowed, but self-review is encouraged to ensure quality.

## Verification Checkpoints
**Track Completion**: Manual verification by the user is required at the end of each feature track.

## Task Lifecycle
1. **Planning**: Define task requirements and success criteria.
2. **Implementation**: Execute the task.
3. **Verification**: Verify implementation meets requirements.
4. **Integration**: Merge/Commit changes.
